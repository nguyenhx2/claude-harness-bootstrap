# The harness in Codex

Codex does not run the skills. It runs the harness the skills produce. Codex reads `AGENTS.md` natively
(root and nested), so no rule conversion is needed. Its hook payload is **identical** to Claude Code's, so
the porter registers the existing hooks directly in `.codex/hooks.json` - no adapter.

## Setup

Scaffold the Claude harness first (the repo needs `.claude/rules/` and `.claude/hooks/`), then port it.
From the repo root:

```bash
python ~/.claude/skills/harness-bootstrap/scripts/port.py --target . --tool codex
```

Requires Python 3. Use `--tool all` to set up Cursor and Codex at once. No release to install; the porter
ships with the skill.

## What lands

| Path | What it does |
|---|---|
| `AGENTS.md` | Already generated by the harness. Codex reads it natively; the porter does not touch it |
| `.codex/hooks.json` | Registers the existing `.claude/hooks/*` under PreToolUse, with matchers for `Bash` and `apply_patch` |

The payload Codex sends (`{cwd, tool_name, tool_input.command}`) matches what the hooks already parse, and
exit 2 denies, exactly as in Claude Code. The `Bash` matcher runs `guard-main-commit`, `check-commit-msg`,
and `protect-secrets`; the `apply_patch` matcher runs `protect-secrets`. Hooks are the same `.sh` (or
`.ps1` on Windows) files; nothing is rewritten.

## What enforces vs. what is advice-only

| Control | Codex matcher | Effect |
|---|---|---|
| `guard-main-commit` | `Bash` | Blocks a direct commit to the default branch |
| `check-commit-msg` | `Bash` | Blocks a non-conventional or AI-attributed commit message |
| `protect-secrets` | `Bash` + `apply_patch` | Blocks a secret read and a patch that touches secret material |
| `protect-adr` | (path lives inside `apply_patch` command) | Best-effort only (see limit) |
| `specs-reminder` | not ported | Not applicable in Codex |
| `agent-history` | not ported | Not applicable in Codex |
| `AGENTS.md` | Read natively | Advice; Codex self-complies with the rules it cannot enforce |

The Bash-based guards run unchanged and are exact.

## Honest limit

Codex routes file edits through `apply_patch` with the path **inside** `tool_input.command`, not in a
`tool_input.file_path` field. `protect-adr` reads a file path, so on Codex it is best-effort - an edit to
an Accepted ADR is not reliably blocked. The Bash guards (secrets, commit target, commit message) are
exact.

## Verify it works

The hook payload is shared with Claude Code, so test a hook directly:

```bash
echo '{"cwd":".","tool_name":"Bash","tool_input":{"command":"git commit -m x"}}' \
  | bash .claude/hooks/guard-main-commit.sh ; echo "exit=$?"    # on main, expect exit=2
```

On Windows use the `.ps1` hook and check `$LASTEXITCODE`. In the tool itself, try to read `.env` or commit
to `main` and confirm each is blocked.
