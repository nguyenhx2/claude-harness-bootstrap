# The harness in Cursor

Cursor does not run the skills. It runs the harness the skills produce. Cursor reads `AGENTS.md` and
`.cursor/rules/*.mdc`; enforcement runs through `.cursor/hooks.json` and a generated adapter that
translates Cursor's hook payload to the shape the existing `.claude/hooks/*.sh` expect, then translates
their exit code back.

## Setup

Scaffold the Claude harness first (the repo needs `.claude/rules/` and `.claude/hooks/`), then port it.
From the repo root:

```bash
python ~/.claude/skills/harness-bootstrap/scripts/port.py --target . --tool cursor
```

Requires Python 3 and `bash` on PATH (the adapter shells out to the `.sh` hooks). No release to install;
the porter ships with the skill.

## What lands

| Path | What it does |
|---|---|
| `.cursor/rules/*.mdc` | One per `.claude/rules/*.md`. A rule with no `paths:` becomes `alwaysApply: true`; `paths: [glob]` becomes `globs: glob` |
| `.cursor/hooks.json` | Registers the adapter on `beforeShellExecution` and `beforeReadFile` |
| `.cursor/hooks/cursor-adapter.py` | Normalizes Cursor's payload to `{cwd, tool_name, tool_input}`, runs the matching `.claude/hooks/*.sh`, and emits `{"permission": "allow"\|"deny", ...}` |
| `AGENTS.md` | Already generated by the harness; Cursor reads it natively (no port step) |

The adapter maps a shell event to the Bash guards (`guard-main-commit`, `check-commit-msg`,
`protect-secrets`) and a read event to `protect-adr` and `protect-secrets`. A hook that exits 2 becomes a
`deny` with the hook's stderr as the user message; otherwise `allow`.

## What enforces vs. what is advice-only

| Control | Cursor event | Effect |
|---|---|---|
| `protect-secrets` | `beforeShellExecution` + `beforeReadFile` | Blocks a secret read (shell or file) |
| `guard-main-commit` | `beforeShellExecution` | Blocks a direct commit to the default branch |
| `check-commit-msg` | `beforeShellExecution` | Blocks a non-conventional or AI-attributed commit message |
| `protect-adr` | `beforeReadFile` path | Blocks on read; the pre-edit block is best-effort (see limit) |
| `specs-reminder` | not ported | Not applicable in Cursor |
| `agent-history` | not ported | Not applicable in Cursor |
| `.cursor/rules/*.mdc` | Loaded by Cursor | Advice only |

Everything reachable through a shell command or a file read blocks the same as in Claude Code.

## Honest limit

Cursor's `afterFileEdit` is observational only. An edit to an Accepted ADR is flagged **after** the fact,
not blocked before it happens. In Claude Code that edit is blocked pre-hoc.

## Verify it works

Run the adapter self-test, which exercises the translation against documented sample payloads:

```bash
python ~/.claude/skills/harness-bootstrap/scripts/port.py --self-test    # expect 5/5
```

It denies `cat .env`, denies a commit to `main`, allows `npm test`, denies reading `.env`, and allows
reading `src`. The same test runs in CI. In the tool itself, try to read `.env` or commit to `main` and
confirm each is denied.
